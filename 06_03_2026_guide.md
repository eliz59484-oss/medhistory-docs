# Гайд по итогам разработки: Аудио-парсинг и Vibe Coding

## Что было сделано сегодня
1. **Настройка инфраструктуры (MCP)**:
   - Supabase (базы данных и хранилище файлов)
   - Fetch (сетевые запросы)
   - GitHub (создание репозиториев и пуш кода)
   - Интеграция ключей `OPENROUTER_API_KEY` (Gemini) и `GROQ_API_KEY` (Whisper).

2. **Стабилизация аудиозаписи на смартфонах**:
   - Полностью удален костыль `window.SpeechRecognition`, который вызывал зависания, дублирование текста и автостарт системных окон на Android.
   - Запись переведена на чистый `MediaRecorder`, идеально работающий как на iOS, так и на Android.

3. **Обход геоблокировок и CORS (маршрутизация API)**:
   - Фронтенд больше не делает запросы к API Groq напрямую. Это вызывало ошибку `failed to fetch` на смартфонах из-за строгих политик безопасности браузеров.
   - Создан мощный серверный шлюз `/api/transcribe`. Теперь телефон передает аудио на наш сервер, а сервер общается с ИИ.
   - Ошибка `403 Forbidden` от Groq решена заменой аудиомодели на доступную `whisper-large-v3-turbo`.
   - **Важно:** поскольку Groq блокирует РФ, для *локальной* разработки необходим включенный VPN (Psiphon/AdGuard) на том компьютере, где запущен сервер Next.js. После публикации (деплоя) приложения эта проблема решится автоматически.

4. **Чистка интерфейса**:
   - Устранен визуальный баг: из формы `PatientForm.tsx` удален дублирующийся блок прикрепления фото/документов.

## Золотые правила Vibe Coding (Памятка)
- План должен быть в `task.md`.
- Антигравити предлагает решения через `implementation_plan.md` — вы только одобряете (Accept).
- Не чините баги руками! Если кнопка не жмется или вылезла ошибка "403" — просто скиньте текст Антигравити. ИИ создан для того, чтобы анализировать логи и находить узкие места.
- Каждый рабочий этап завершается коммитом и пушем в GitHub — так код никогда не потеряется.
