## 📐 Три способа создания дизайна

### Способ 1: Описание словами
> Открываете Antigravity → описываете, как должен выглядеть дизайн → получаете шаблонный, но рабочий результат.

**Плюсы:** Быстро, просто  
**Минусы:** Результат типовой, «как у всех»

### Способ 2: Через исследование (Deep Research)
> Используете Gemini / GPT / Perplexity → просите исследовать рынок, найти сайты конкурентов, оценить их дизайн → получаете ТЗ на основе анализа.

**Что запрашивать:**
- Дизайн конкурентов
- Гайдлайны, шрифты, цветовые решения
- Информацию о вашем продукте и аудитории

### Способ 3: На основе готового кода (⭐ рекомендуется)
> Уже есть рабочий продукт с каким-то интерфейсом → просите ИИ сделать крутой дизайн на основе существующего функционала.

**Это самый частый сценарий** — когда продукт работает, но выглядит базово.

## 🎨 Построение лендинга: Пошаговый процесс

### Шаг 1: Изучение проекта
```
Запрос: "Изучи проект, весь функционал. Расскажи, что может 
этот сервис и для какой аудитории."
```
> Нейронка анализирует код, понимает продукт, формирует контекст для дальнейшей работы.

### Шаг 2: Проработка структуры
```
Запрос: "Буду разрабатывать дизайн главной страницы. 
Сейчас там только логин и пароль. Пока не пиши код — 
давай проработаем структуру."
```

**Типовая структура лендинга:**
1. 🔝 **Шапка (Header)** — логотип, навигация, CTA-кнопка
2. 🏠 **Главный экран (Hero)** — заголовок, подзаголовок, CTA
3. ✅ **Блок доверия** — цифры, статистика ("200 000+ запросов")
4. 💪 **Преимущества** — почему выбрать вас
5. ⚡ **Основные фичи** — ключевой функционал продукта
6. 📸 **Демонстрация** — скриншоты/видео приложения
7. 💰 **Тарифы** — если есть (можно пропустить, если уже в приложении)
8. 📝 **CTA + регистрация** — финальный призыв к действию

### Шаг 3: Тексты по Jobs to be Done
```
Запрос: "Хочу, чтобы текст на главной странице был написан 
по методологии Jobs to be Done."
```
> JTBD ориентируется на **решение проблем пользователей**, а не на свойства продукта. Тексты получаются убедительнее.

### Шаг 4: Референсы
```
Запрос: "Изучи сайт-референс [ссылка] и сделай лендинг 
с таким же дизайном."
```
Или: кидаете **скриншот** понравившегося сайта.

### Шаг 5: Дополнительный промпт для качества
```
"Сделай иерархию акцентов. Соблюдай все правила UX и UI."
```

## 🎯 Иерархия акцентов — главное правило дизайна

> **Где подсветить — там и внимание. Всё.**

- Жирный текст = главный акцент
- Размер шрифта определяет важность
- Не нужны теории про «F-паттерн чтения» или «боли»
- Просто: **что выделено — то и видит пользователь**

```
Промпт: "Соблюдай иерархию акцентов" — и нейронка 
правильно расставит визуальные приоритеты.
```

## 🛠️ Инструменты для дизайна

### Google AI Studio (⭐ для фронтенда)
- Адрес: aistudio.google.com
- **Входить под тем же аккаунтом**, что и Antigravity (Ultra-тариф)
- Выбрать фреймворк: **Next.js** (легко переносится в Antigravity)
- Результат качественнее, чем в Antigravity, но дольше генерирует
- Можно **скачать код** и закинуть в проект

**Перенос из Google AI Studio → Antigravity:**
1. Скачать код (кнопка "Code" → Download)
2. Разархивировать в папку проекта
3. Сказать Antigravity: *"Вот разработанный лендинг. Замени текущую страницу на этот."*

### Сайты-референсы

| Ресурс | Для чего |
|--------|----------|
| **21st.dev** | Готовые UI-компоненты с промптами (копируешь промпт → вставляешь в Antigravity) |
| **Pinterest** | Поиск визуальных референсов лендингов |
| **Lovable** | Альтернатива Google AI Studio |

### Парсинг чужих сайтов
- Существуют HTML-парсеры, которые копируют всю структуру сайта
- Картинки, анимации, код — всё скачивается
- Результат требует доработки, но даёт хорошую базу
- Можно просто в Antigravity: *"Зайди на [сайт], подбери нужные элементы и поставь на мой лендинг"*

## 🌿 GitHub: Работа в команде через ветки (Branches)

### Зачем ветки?
- **Безопасность** — никто не сломает основной код
- **Параллельная работа** — один делает фронтенд, другой бэкенд
- **Проверка перед слиянием** — главный разработчик ревьюит изменения

### Схема работы:
```
Main (основная ветка)
  ├── Разработчик 1 работает с бэкендом → push в main
  └── Разработчик 2 → создаёт ветку "site"
       → делает фронтенд
       → push в ветку "site"  
       → Главный проверяет → Merge в main ✅
```

### Команды:
```
Запрос Antigravity: "Создай отдельную ветку в репозитории. 
Назови её 'site'. Добавь в неё [изменения]."

После завершения: "Отправь код в ветку 'site'."
```

## 🔗 Claude Code: Мостик через Telegram

- Можно прокинуть мостик из терминала Claude Code в Telegram
- **Не нужны API, не нужно тратить токены**
- Всё дублируется: терминал ↔ Telegram
- Можно подтверждать команды (Run/Reject) прямо с телефона
- **Кейс:** На встрече с инвестором → идея → промпт с телефона → реализация

## 📊 Бизнес-цикл продукта

```
Исследование → Гипотеза → Разработка → Маркетинг → Продажа
      ↑                                                    |
      └────────── Обратная связь от юзеров ←───────────────┘
```

### 3 типа исследований:
1. **Качественное** — личное общение с аудиторией (нельзя делегировать!)
2. **Количественное** — анкеты (не более 3-4 вопросов)
3. **Онлайн-разведка (OSINT)** — добыча информации из открытых источников

### Статистика гипотез:
- **1 из 10** идей выстреливает (базовая)
- **3 из 100** при масштабных данных
- Стоимость гипотезы: раньше ~1 млн ₽, сейчас ~300-500 тыс. ₽
- **Уметь отказаться от идеи** — тоже экономия

## 💡 Разное (важные заметки)

- **Skills:** Описание скилла (description) критично — от него зависит, подхватит ли агент нужный скилл автоматически
- **Модели:** Можно менять через `/model` в Antigravity
- **Локальные модели:** Qwen 3.5 можно запустить на Mac Mini — работает ночью, пока вы спите, экономит токены
- **Antigravity vs Claude Code:** В тестах Antigravity работал быстрее при одинаковых промптах
- **Контекст:** При зависании чата → новый чат → *"Изучи файлы проекта и посмотри предыдущий чат"* → подхватывает контекст
- **Будущее приложений:** Claude Code может заменить многие отдельные приложения (утилиты, инструменты)
