## 1. Wireframing (Вайрфрейминг) — Блок-схемы продукта

### Что это такое
- **Wireframing** — это полноценная схема продукта, которая не просто визуально показывает связи, но и детально описывает:
  - Откуда идут входные данные
  - Куда они попадают
  - Какой триггер срабатывает
- Отличается от обычных блок-схем (типа N8N) бо́льшей детализацией

### Место в процессе разработки
Wireframing идёт **между идеей и техническим заданием**:

```
Идея → Wireframing (блок-схема) → Техническое задание → Архитектура → Разработка
```

### Почему это критически важно

> **Без качественной документации срок разработки увеличивается на 6–8 месяцев** (даже с вайб-кодингом — потеряете минимум 2 месяца).

- Нейронки сами рисуют схемы, но они, как правило, содержат ошибки
- Когда схема хорошо разрисована — всё становится чётким, ошибок меньше
- Wireframing можно **прикреплять к договорам** с заказчиками — не будет споров
- Перед каждым проектом/задачей нужно рисовать блок-схему

## 2. Практический пример: Telegram-бот (помощник по обучению)

### Идея бота
Telegram-бот с двумя режимами:
1. **База знаний** — хранит лекции, отвечает на вопросы по ним
2. **Исследование** — парсит внешние источники (YouTube, Telegram-каналы, веб) и присылает сводки

### Построение блок-схемы

**Правило:** всё начинается с **триггера** (кнопка Start, команда, вызов внешней системы, таймер).

Структура схемы бота:

```
Триггер (Start) → Приветственное сообщение (текст + картинка + inline-кнопки)
                      ├── 🔍 Исследовать
                      └── 📚 База знаний (лекции)

Запрос юзера → Агент (Системный промпт + Память + LLM)
                  ├── Perplexity → ответ в PDF
                  ├── Telegram-каналы (список ссылок)
                  └── YouTube (по фильтрам через MCP)
```

**Ключевые элементы агента:**
- Системный промпт (фильтр)
- Память (на базе Supabase)
- LLM модель (через OpenRouter)

## 3. Технологический стек

### Зачем знать стек
- Единственное, на что можно реально повлиять в разработке — **технологический стек**
- Знание стека позволяет решать проблемы, которые разработчики считают нерешаемыми

### Кейс: Приложение распознавания объектов
- Задача: сфотографировать любой объект → AI определяет, что это, где находится, история
- Проблема: ни Gemini, ни GPT не справлялись с точным определением
- Решение: **SERP API** — даёт доступ к функции «поиска по фото» от Google/Яндекс
- Результат: 10 из 10 правильных определений

> **Вывод:** Следите за технологическим стеком, читайте новости, настройте бота для отслеживания новинок.

### Стек для учебного бота

| Компонент | Технология |
|-----------|-----------|
| Мессенджер | Telegram Bot API |
| Парсер | **Firecrawl** (лучший на текущий момент) |
| База данных | **Supabase** (90% случаев — лучший выбор) |
| LLM | **Gemini Flash** через **OpenRouter** |
| Поисковик | **Perplexity** (глубокое исследование) |
| Язык | **Python** |
| Протокол | **MCP** (Model Context Protocol) |

### Рекомендации по выбору БД
- **Supabase** — для 90% проектов (создана специально для AI, лёгкое подключение к RAG)
- **PostgreSQL (через SQL)** — для масштабируемых проектов с раздутой базой
- Не рекомендуется: Google Sheets, Notion и другие «костыли»

## 4. MCP (Model Context Protocol) — подключение сервисов

### Что даёт MCP
- **Один раз подключил — забыл**: не нужно заходить в сервисы вручную
- Ускоряет разработку **в 10 раз**
- Автоматически работает во всех проектах
- Подключается глобально в Antigravity (и в Code-Code тоже)

### Примеры без MCP vs с MCP

| Без MCP | С MCP |
|---------|-------|
| Зайти в GitHub → создать репо → скопировать ссылку → отправить AI | Сказать: «Отправь в репозиторий» |
| Зайти в Supabase → создать проект → настроить таблицы | Сказать: «Создай проект в Supabase» |

### Подключённые MCP серверы

| Сервер | Инструментов | Назначение |
|--------|-------------|-----------|
| **Supabase** | 29 | База данных, миграции, таблицы |
| **GitHub** | ~20 | Репозитории, деплой кода |
| **Firecrawl** | 12 | Парсинг веб-страниц |
| **Perplexity** | 1 | Поиск и исследование |
| **REST API** | — | Универсальное подключение к API |

### Как подключать MCP
1. **Через магазин** (3 точки → MCP Server → поиск → Install)
2. **Вручную** — попросить AI: «Подключи [название] MCP» → он сам соберёт JSON-конфиг

### Где хранятся MCP настройки
В JSON-файле конфигурации (виден в файловой директории проекта). Каждый MCP — небольшой кусочек кода с токеном.

## 5. Context7 — актуальная документация

### Проблема
- Нейронки работают на устаревших данных (дата обучения отстаёт)
- API сервисов меняются, появляются ошибки, которые AI не может найти

### Решение — Context7
- Документация обновляется **в реальном времени**
- AI использует актуальную информацию вместо устаревшей

### Пример проблемы
Сервис писал, что есть бесплатная квота 1500 запросов. AI верил этому. На деле квоту убрали полгода назад → бот зацикливался. С Context7 такого бы не произошло.

## 6. Глобальные правила (Global Rules)

### Как установить
3 точки → Customization → Global → вставить промпт

### Что даёт
- Правила работают во всех чатах Antigravity
- Можно прописать: «Всегда задавай вопросы при разработке»
- Промпт с правилами разработки был выслан в группу

## 7. Работа с документацией — итоговый чеклист

### Перед стартом проекта:
- [ ] Написать идею
- [ ] Нарисовать Wireframing (блок-схему) в Холсте
- [ ] Определить технологический стек
- [ ] Подключить все MCP серверы
- [ ] Подготовить документацию/ссылки на сервисы без MCP
- [ ] Добавить материалы (лекции, данные) в файловую директорию
- [ ] Установить глобальные правила
- [ ] Сделать перепроверку: попросить AI описать идею для подтверждения понимания

### При работе с непопулярными сервисами:
- Если MCP нет в магазине → отправить ссылку на документацию вручную
- AI ознакомится и будет разрабатывать правильно

## 8. Полезные термины

| Термин | Значение |
|--------|----------|
| **Wireframing** | Детальная блок-схема продукта с описанием данных и триггеров |
| **RAG** | Методология хранения данных и поиска информации нейронкой |
| **Chunks (чанки)** | Кусок законченной информации (текст, словосочетание) |
| **Embeddings (эмбеддинги)** | Функция для работы с данными, векторное представление текста |
| **MCP** | Model Context Protocol — стандарт подключения AI к внешним сервисам |
| **Context7** | Сервис актуальной документации в реальном времени |
| **Column Function** | Функция для управления ботом через текст вместо кнопок |
| **SERP API** | Сервис доступа к поисковым функциям Google/Яндекс |
