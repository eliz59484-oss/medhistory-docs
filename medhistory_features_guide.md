# Гайд по функционалу MedHistory (Спецификация)

## 📌 О проекте
**MedHistory** — это прогрессивное веб-приложение (PWA) для врачей, автоматизирующее процесс сбора, анализа и оформления медицинского анамнеза с помощью передовых нейросетей.

---

## 🚀 Текущий рабочий функционал (MVP)

### 1. Голосовой ввод и транскрибация (Whisper-режим)
Врач может надиктовать мысли или результаты осмотра в свободной форме.
- **Технология:** Запись аудио через `MediaRecorder` с проксированием через локальный сервер (Next.js `/api/transcribe`) для обхода геоблокировок и политик CORS.
- **Движок:** Groq API (модель `whisper-large-v3-turbo`).
- **Преимущества:** Высокая скорость, идеальное распознавание медицинских терминов на русском языке, стабильная работа как на компьютере, так и на смартфонах (iOS / Android).

### 2. Умный ИИ-разбор сырого текста (Gemini-анализ)
Сплошной поток текста (как продиктованный голосом, так и написанный вручную) автоматически раскладывается по структурированным полям медицинской карты.
- **Технология:** Google Gemini 2.5 Flash через OpenRouter.
- **Функция:** Извлечение структурированных данных:
  - Жалобы пациента
  - Анамнез заболевания (Anamnesis morbi)
  - Анамнез жизни (Anamnesis vitae)
  - Объективный статус
  - Диагноз (предварительный/клинический)
  - Рекомендации и назначения

### 3. Специализированные шаблоны врачей (Промпты)
ИИ понимает контекст и специфику разных врачей. В зависимости от выбранного профиля, нейросеть применяет разные правила извлечения и оформления текста.
**Реализованные профили:**
- Врач общей практики (Терапевт)
- Педиатр
- Хирург
- Гинеколог
- Проктолог
- Кардиолог

### 4. Мультиплатформенная форма пациента `PatientForm.tsx`
Единый и удобный интерфейс с чистым UI без дублей, оптимизированный под работу пальцем на мобильных устройствах. Включает в себя:
- Выбор медицинского профиля
- Блок диктовки с визуальной индикацией
- Редактируемые текстовые поля с результатами ИИ-разбора
- Блок прикрепления медицинских документов (UI модуль `PhotoUploader`)

## 🚧 Планы на ближайшие релизы (Roadmap)

### Этап 1: Безопасность и Деплой (Ближайший релиз)
1. **Защита данных:** Подключение защищенного контура авторизации (Login/Password) с помощью Supabase Auth. Медицинские данные будут закрыты от случайных глаз.
2. **Публичный Деплой:** Размещение проекта на облачном Vercel. Доступ к приложению по стабильному `https://` домену без необходимости держать запущенным локальный компьютер. Устранение нужды в VPN на телефоне.

### Этап 2: Расширение функционала (Новые возможности)
1. **Профиль "Врач функциональной диагностики":** 
   - Добавление нового системного промпта.
   - Настройка специфических полей (ЭКГ, ЭхоКГ, Холтер, СМАД, УЗИ и т.д.).
2. **Экспорт в MS Word (`.docx`):**
   - Генерация красивого, отформатированного медицинского заключения по стандартам ГОСТ.
   - Скачивание готового файла в один клик для печати и выдачи пациенту.
3. **Хранение данных и База пациентов:**
   - Полноценная привязка базы данных Supabase.
   - Dashboard с архивом созданных карточек пациентов с возможностью поиска.
   - Сохранение прикрепленных фотографий/анализов в облачное хранилище Supabase Storage.
