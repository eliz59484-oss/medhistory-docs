## 1. Блок вопросов и проблем участников

### Проблемы при работе
- **Терминал зависает на команде `run`** — частая проблема участников. Решается несколькими способами: перезапуск, смена модели, команды из чата.
- **Vercel не принимает обновления** — рекомендация: если приложение с базой данных, лучше деплоить не на Vercel, а на **Timeweb** или аналоги. Vercel больше подходит для фронтенд-проектов без тяжёлой серверной логики.
- **Кириллица и shell-проблемы на Windows** — у части участников решилось, рекомендуется проверить инструкции из общего чата.

### Вопросы по API и моделям
- **Выбор модели через OpenRouter:** Просто говорим агенту словами: «Хочу, чтобы работал Opus 4.6» — он сам переключает.
- **Температура модели:** Настраивается программно (в коде), а не в интерфейсе OpenRouter.
  - **Высокая температура** → максимальная креативность, свободная генерация.
  - **Низкая температура** → точное следование инструкциям, работа с документами.
  - Георгий рекомендует **лучше прописать хороший промпт**, чем крутить температуру.
- **Whisper и TTS:** Whisper отсутствует в OpenRouter. Лучшая бесплатная альтернатива — **Groq** (требует VPN). Яндекс SpeechKit — платный и дорогой. Можно комбинировать: основная модель на OpenRouter, а TTS/STT — из OpenAI отдельно.

### Статус участников
- Часть группы опережает программу (уже ведут свои проекты).
- Часть отстаёт — Георгий напоминает: домашнее задание — залить на GitHub текущий проект, ТЗ, архитектуру, либо добавить скиллы.
- Один из участников решил проблему зависания, открыв **два диалоговых окна** в одном проекте — когда агент завис в первом, продолжил во втором.

## 2. Что такое Claude Code

> **Claude Code** — это продукт от Anthropic, работающий из терминала. Аналог Antigravity, но с бо́льшими возможностями для кастомизации.

### Ключевые отличия от Antigravity

| Параметр | Antigravity | Claude Code |
|----------|-------------|-------------|
| Интерфейс | Визуальный агент в IDE | Терминал |
| Модель | Opus 4.6 | Opus 4.6 (та же) |
| Кнопка `run` | Нужна для каждого шага | Не нужна — всё автоматически |
| Скиллы | Добавляются, но не всегда подхватываются | Автоматически подхватываются |
| Мультиагентность | Ограниченная | Полноценная (рой агентов) |
| Откат изменений | Есть кнопка reject | Нет — изменения безвозвратны |
| Безопасность | Контролируется IDE | Требует внимательности |
| Подписка | Фиксированная | $20/мес (Pro) или $100/мес (без лимитов) |

### Когда что использовать
- **Antigravity** — готовый сервис для разработки. Экономичнее. Хорош для новичков.
- **Claude Code** — для продвинутой работы: мультиагентность, бесконечные скиллы, автоматизация, управление с телефона.
- Можно использовать **оба одновременно**: Antigravity разрабатывает, Claude Code — проверяет (аудитор/ревьюер).

> **Важно:** При работе с Claude Code нужно чётко ставить задачи. Если неправильно сформулировать запрос, он может переделать весь проект, и откатить будет невозможно. Пример: если хотите просто спросить — пишите «Не пиши код, объясни мне...», иначе он начнёт менять файлы.

## 3. Мультиагентная система

Главное обновление Claude Code (вышло **3 марта 2026**) — полноценная мультиагентная система.

### Как это работает
- Даёте одну задачу → создаётся рой агентов (2, 3, 5, 10, 15...).
- Каждый агент работает **параллельно**, в своей сессии.
- Агенты не видят чаты друг друга — у каждого свой контекст.
- По завершении приходят с результатами в основной чат.

### Пример из демонстрации (5 агентов)
```
Запрос: «Запусти 5 агентов для моего проекта»

Агент 1 → Поиск потенциальных багов
Агент 2 → Оценка процента покрытия кода
Агент 3 → Описание функционала проекта
Агент 4 → Анализ того, что нужно доработать
Агент 5 → Оценка проекта как full-stack разработчик
```
Все 5 запустились параллельно, быстро просканировали проект и вернули структурированные отчёты.

### Практическое применение
- Один агент — фронтенд, другой — бэкенд, третий — тексты, четвёртый — SEO.
- Можно создать «начальника отдела» (главного агента) с командой субагентов — и давать задачи ему.

## 4. Скиллы: два уровня

### Уровень 1 — Улучшение базовых навыков
Набор текстовых инструкций (MD-файл), который заставляет Claude Code лучше выполнять определённые задачи.

**Особенности:**
- Если Opus 4.6 справляется с задачей **без скилла** — он его не трогает.
- Если скилл не используется — Claude Code может **удалить его сам**.
- При выходе Opus 4.7 / 5.0 — часть скиллов отвалится автоматически (уже не нужны).

### Уровень 2 — Самообучающийся скилл
- Вы выполняете задачу, потом даёте обратную связь: «Мне не нравится, сделай лучше вот здесь».
- Скилл учитывает замечания и **обновляет себя**.
- С каждой итерацией работает лучше.
- На 10-20 повторение — задача выполняется **идеально**.

**Версионирование скиллов:**
```
Версия 1.0 → Базовый скилл
Версия 1.1 → Добавлена обратная связь по отступам
Версия 1.2 → Улучшены заголовки
Версия 1.3 → Исправлена работа со шрифтами
```

> **Применение:** Если у вас есть регулярные задачи (презентации, коммерческие предложения, проверки кода) — натренируйте скилл, и он будет выполнять их всё лучше.

## 5. Skill Creator

Встроенный инструмент Claude Code для **автоматического создания скиллов**.

### Как работает
1. Вызывается через `/skill-creator` в Claude Code.
2. Вы описываете, какой скилл нужен.
3. Skill Creator создаёт **сразу 5 вариантов** скилла.
4. **Тестирует** каждый на реальной задаче.
5. **Сравнивает** результаты (включая работу без скилла).
6. Выставляет **баллы** и выбирает лучший.
7. Согласовывает результат с вами.

> По сути — **встроенный A/B-тест** для скиллов.

### Установка Skill Creator
```
1. В Claude Code нажать /
2. Перейти в Plugins → Search → "skill"
3. Найти Skill Creator
4. Установить глобально (user scope)
5. Перезапустить терминал
```

## 6. MCP — Model Context Protocol

> **MCP** — это метод быстрого подключения к внешним сервисам. Никакой магии, просто протокол интеграции.

### Проблема, которую решает MCP
Документация сервисов постоянно обновляется. Вы написали код по версии 1 — через неделю API изменилось, код не работает. Приходится вручную разбираться.

### Решение: Context7 (MCP-сервер)
- Собрана **вся актуальная документация** популярных сервисов.
- Агент **перед написанием кода** автоматически проверяет документацию.
- При следующем обновлении — снова проверит, ничего ли не изменилось.
- Результат: у вас **всегда свежая документация**, голова не болит.

### Другие MCP-примеры
- **GitHub MCP** — публикация проекта за секунду (вместо ручного API).
- Маркетплейсы MCP — сотни готовых подключений.

## 7. Скиллы vs Агенты

| Скиллы (Skills) | Агенты / Субагенты (Agents) |
|---|---|
| Текстовый файл (MD) | Контейнер с инструментами |
| Пошаговые инструкции | Инструкции + API + MCP + коннекторы |
| Простые в создании | Сложнее, но мощнее |
| Улучшают одну операцию | Решают комплексную задачу целиком |

**Пример агента:** Готовый агент для анализа ассоциированных конверсий — уже знает, куда подключиться, какие метрики сканировать, какие методы использовать. Скачал → развернул → работает с минимумом настроек.

**Маркетплейсы скиллов и агентов:**
- Уже **400+ тысяч скиллов** на различных маркетплейсах.
- Подходят для Claude Code, OpenClaw и Antigravity.
- ⚠️ **Проверяйте звёзды и профиль автора** перед установкой — риск вредоносного содержимого.

## 8. Глобальные и локальные скиллы

### Глобальные (тильда `~` в пути)
- Работают **во всех проектах**.
- Подходят для универсальных задач: аудит кода, проверка безопасности.
- Путь: `~/.claude/skills/...`

### Локальные
- Работают **только в текущем проекте**.
- Подходят для специализированных задач: дизайн контент-завода, проверка фото/видео.
- Путь: `./project/.claude/skills/...`

### Рекомендация Георгия
- **Глобально** — универсальные скиллы (код-ревью, безопасность, аудит).
- **Локально** — специфичные для проекта (дизайн, контент, отрасль).
- Причина: глобальный дизайнерский скилл может случайно включиться в маркетинговом проекте.

## 9. Практика: установка и запуск

### Установка Claude Code (на примере Сергея)

**Шаг 1.** Открыть Antigravity, написать: *«Помоги мне развернуть Claude Code в терминале»*.

**Шаг 2.** Последовательно выполнить команды:
```bash
# Установка Node.js (если нет)
# Установка Claude Code глобально
# Авторизация через аккаунт с подпиской
```

**Шаг 3.** Авторизация:
- Выбираем **вариант 1** (подписка) — безлимитное использование.
- ⚠️ Войти именно тем аккаунтом, где **оплачена подписка** Claude.

**Шаг 4.** Расположение:
- Рекомендуется поместить Claude Code **справа** (рядом с Antigravity).
- Правая кнопка → Panel Position → Right.

### Запуск нового экземпляра
В терминале просто пишете `claude` — и запускается новый агент. Можно открыть несколько экземпляров для разных проектов.

### Смена модели
```
/ → model → выбрать Opus стрелками → Enter
```

## 10. Голосовое управление (Gladia)

Отдельный сервис для голосового ввода текста в любое поле — в Antigravity, Claude Code, браузер и т.д.

### Как работает
1. Установили Gladia.
2. Нажимаете горячую клавишу → появляется зелёный индикатор.
3. Говорите текст голосом.
4. Отпускаете — текст вставляется в активное поле.

### Стоимость
- Подписка ~$20/мес.
- Есть бесплатный период.
- Можно разработать аналог самостоятельно (~1 час работы).

## 11. Экономика и выбор инструмента

| Инструмент | Стоимость | Лимиты | Комментарий |
|---|---|---|---|
| **Antigravity** | Подписка (фикс.) | Щедрые | Самый экономичный для разработки |
| **Claude Code Pro** | ~$20/мес | Быстро кончаются | Хватает для базовой работы |
| **Claude Code Max** | ~$100/мес | Безлимит | Для серьёзной работы |
| **OpenClaw + Kimi 2.5** | Копейки | — | Аналог Opus, в 10 раз дешевле |

### OpenClaw — дешёвая альтернатива
- В 10 раз дешевле Claude Code.
- Поддерживает **Kimi 2.5** (аналог Opus 4.6 за копейки).
- **Минусы:** менее безопасный, может удалить файлы; крупные компании (Google, Anthropic) банят его ключи.
- GPT работает с OpenClaw (OpenAI выкупили их).

### Рекомендация
- Для работы с компьютера — **подписка** (ничего не тратится дополнительно).
- Для работы с телефона (Telegram-бот и т.д.) — **токены** (они дорогие у Opus).
- Главный навык для экономии — **правильно писать скиллы и запросы**.
