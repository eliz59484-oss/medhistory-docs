# Экосистема Antigravity и Vibe Coding: Гайд для Разработчиков

## 🌌 Что такое Antigravity?
**Antigravity** — это десктопная среда разработки и агентская экосистема, разработанная на базе глубокой интеграции ИИ. Это не просто автодополнение кода, а полноценный "второй пилот", который способен самостоятельно анализировать архитектуру, предлагать решения, писать код, дебажить ошибки и управлять инфраструктурой проекта (базы данных, GitHub, серверы) от лица пользователя.

Ключевая идея Antigravity — это переход от классического, ручного программирования к методологии **Vibe Coding**.

---

## 🎶 Методология Vibe Coding (Программирование по вайбу)
Vibe Coding — это парадигма разработки, при которой человек-разработчик выступает в роли **Архитектора и Режиссёра**, а ИИ-Агент выступает в роли **Исполнителя (Программиста)**.

### Основные принципы:
1. **Общение на естественном языке:** Разработчик формулирует бизнес-логику и требования человеческим языком. ИИ переводит это в код.
2. **Отложенный контроль:** Разработчик не смотрит на каждую строчку сгенерированного кода, а оценивает результат работы (UI, функционал, логи).
3. **Промпт-Инженерия > Синтаксис:** Умение правильно поставить задачу агенту становится важнее, чем знание специфического синтаксиса фреймворка.
4. **Agentic Actions (Агентские действия):** ИИ не просто выдает куски кода, а сам создает файлы, устанавливает зависимости в терминале, перезапускает сервер, ищет файлы по рабочему пространству и фиксирует изменения в систему контроля версий (Git).

## 🛠 Инфраструктура и Инструменты под капотом

Сила Antigravity в том, что ИИ имеет доступ ко множеству специализированных инструментов (Tools) и Протоколу Контекста Моделей (MCP).

### 1. Model Context Protocol (MCP) Скиллы
Antigravity использует MCP-серверы для расширения возможностей базовой нейросети:
- **`@modelcontextprotocol/server-github`**: Позволяет агенту читать репозитории, создавать новые, делать коммиты и управлять PR (Pull Requests) напрямую через API GitHub, используя `Personal Access Token` пользователя.
- **`@supabase/mcp-server-supabase`**: Интеграция с базой данных Supabase. Агент может генерировать SQL-скрипты миграций, создавать таблицы для пользователей, настраивать Storage (для файлов) и управлять RLS (Row Level Security).
- **`@modelcontextprotocol/server-fetch`**: Способность ИИ ходить в интернет, запрашивать данные по HTTP и тестировать API без привлечения пользователя.

### 2. Системные инструменты агента (Built-in Tools)
- **Терминал (Bash/PowerShell):** Агент запускает команды `npm run dev`, `git add`, `node`, `grep` — анализируя их вывод и самостоятельно решая ошибки "на лету" (self-reflection).
- **Файловая система (`view_file`, `write_to_file`):** Прямое чтение и модификация кода. Агент не просто пишет код в чат, он внедряет его (Patch / Replace) прямо в файлы проекта.
- **Task Boundaries (Управление задачами):** Агент способен создавать чек-листы (`task.md`) и планы внедрения (`implementation_plan.md`), разбивая большую цель на микро-шаги и синхронизируя свой статус с пользователем.

## 🧠 Как работать с Antigravity максимально эффективно?

### 1. Держите ИИ в контексте
Когда возникает ошибка (например, в браузере), самый быстрый способ ее решить:
1. Скопировать текст ошибки (или скриншот).
2. Написать в чат контекст: *"Я нажал на кнопку записи, вылетела вот эта ошибка в консоли Chrome"*.
3. **Дать агенту свободу:** *"Проанализируй проблему и предложи план исправления"*.

Агент сам найдет в файлах то место, где падает код, предложит обновленный `implementation_plan.md` и после вашего "ОК" применит патч.

### 2. Сверяйтесь с планами
Любая сложная задача начинается с фазы `PLANNING` (Планирование). Агент создает `implementation_plan.md`, в котором разжевывает:
- В чем причина бага.
- Какие файлы будут изменены (`[MODIFY]`, `[NEW]`).
- Как мы будем проверять, что это сработало (`Verification Plan`).

Если план выглядит логично — просто дайте команду `"Accept"`.

### 3. Релизы и Чекпоинты
Каждая закрытая фича должна сопровождаться **сохранением состояния (Commit)**. Агент умеет делать это сам. Если фича доделана, скажите: *"Отличная работа, сделай Git коммит с осмысленным описанием"*. Это спасает проекты от того, чтобы случайно не сломать то, что уже работало.
